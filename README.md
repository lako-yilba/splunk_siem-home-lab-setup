# 🛡️ Splunk SIEM Home Lab (pfSense + Suricata + Windows + Kali)

## 📌 Overview

This project demonstrates the implementation of a **Security Information and Event Management (SIEM) home lab** using **Splunk Enterprise running on an Ubuntu Server** as the central log management and analysis platform.

The lab environment simulates a **SOC (Security Operations Center)** where logs from multiple systems are centralized, monitored, and analyzed for security events.

The network is protected using **pfSense as the firewall** and **Suricata as the Intrusion Detection System (IDS)**. Security and system logs generated across the environment are forwarded to the **Splunk server hosted on Ubuntu**, where they are indexed and analyzed.

Logs are collected from multiple sources including endpoints, firewall logs, and IDS alerts, allowing security analysts to monitor activity, detect threats, and investigate suspicious behavior.

---

## 🏗️ Architecture

The lab architecture consists of a centralized **Splunk Enterprise server hosted on Ubuntu**, which collects logs from different systems in the network.

### 🔐 Network Security Components

- 🔥 **pfSense Firewall** – Used as the network firewall to monitor and control incoming and outgoing traffic.
- 🚨 **Suricata IDS** – Deployed on pfSense to detect suspicious network activity and intrusion attempts.

---

## 📡 Log Sources

- 🖥️ **Windows Endpoint** → Logs forwarded using **Splunk Universal Forwarder**
- 🐉 **Kali Linux** → System logs forwarded to Splunk
- 🔥 **pfSense Firewall** → Firewall and network logs sent via **Syslog**
- 🚨 **Suricata IDS** → Intrusion detection alerts forwarded to Splunk
- 🗄️ **Splunk Server (Ubuntu)** → Central log collection, indexing, and analysis

---

## 🔄 Log Flow

Windows Endpoint → Splunk Universal Forwarder → 🗄️ Splunk Server (Ubuntu)  
Kali Linux → Syslog / Forwarder → 🗄️ Splunk Server (Ubuntu)  
pfSense Firewall → Syslog → 🗄️ Splunk Server (Ubuntu)  
Suricata IDS Alerts → Syslog → 🗄️ Splunk Server (Ubuntu)

All logs are indexed within Splunk and can be analyzed using **search queries, dashboards, and alerts**.

---

## 🔍 Monitoring and Detection

Using Splunk, security analysts can monitor:

- 🚨 Intrusion alerts generated by **Suricata IDS**
- 🔥 Firewall events from **pfSense**
- 🖥️ Windows security events
- 🐉 Linux system logs
- 📡 Network activity and suspicious connections
